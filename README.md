# arzamaskov.link

## Процесс работы с ветками

### Ветка main:

Это основная стабильная ветка, содержащая проверенный и готовый для продакшена код.  
Изменения вливаются в main только после того, как они были протестированы и стабилизированы в develop.  
Ветка используется для развертывания приложения на сервере через GitHub Actions.

### Ветка develop:

Эта ветка используется для активной разработки. В ней могут быть не полностью стабильные изменения, так как они могут находиться на стадии тестирования.  
Feature-ветки (например, feature/jwt-auth) создаются на основе develop и после завершения работы сливаются в неё.

### Feature-ветки:

Для каждой фичи создаются ветки с именами вроде feature/<название-фичи>, которые создаются на основе develop.  
После завершения работы и тестирования фичи она сливается обратно в develop.  

## Процесс слияния в main:

Когда код в develop считается готовым для релиза, создаётся Pull Request для слияния изменений из develop в main.
После успешного слияния в main, GitHub Actions автоматически запускает процесс деплоя.
